## Welcome to The Services Vault

Every service in my studio has a method behind it. The Services Vault is all of them: the eight companion guides that turn what I do *for* clients into something you can run *yourself*, on your own timeline, without a retainer.

These are not summaries. Each one is the actual framework: the briefs, templates, worked examples, checklists, and decision points. The same thinking that powers client work is written out so you can do the work with your own hands.

This guide turns the bundle into a usable system. It tells you what each guide is for, how they connect, what order to use them in, and how to move from "I bought the methods" to "I ran the method."

> Conviction and competence can live in the same body. You bring the calling; these give you the craft.

## How To Use This Bundle

Do not read it like a novel. Use it like a studio shelf.

Start with the problem in front of you. If the website is the problem, open The Build. If the story is the problem, open The Rewrite. If you cannot figure out the problem, open Before the Session. If you have copy but it is soft, open The Edit.

The bundle works because the guides overlap on purpose. Real projects overlap. A website needs positioning. A LinkedIn rewrite needs voice. A content strategy needs a point of view. AI prompts need judgment before they need syntax.

Use the overlap. That is where the method gets strong.

## The Fast Diagnostic

Before you open a guide, answer this:

**What job am I trying to finish?**

- I need a website that says the right thing.
- I need my professional story to match who I am now.
- I need a content system I can keep.
- I need to sound like myself when someone else writes for me.
- I need to edit what I already wrote.
- I need to reposition the brand before I write another page.
- I need to prepare for a strategy conversation.
- I need AI to draft with my voice instead of flattening it.

Your answer tells you where to begin. A bundle only works when it becomes a choice, not a pile.

## If You Are Building A Website

Start with **The Build Copy Guide**.

Do not start with colors. Do not start with a template. Do not start with "About" because it feels personal and therefore easier. Start with the job of each page.

The Build teaches the structure: homepage, services pages, About, contact, proof, calls to action, and the line-level test that keeps copy from becoming decoration. It gives you the architecture before the sentences, because sentences without architecture turn into a brochure.

After you draft, use **The Edit** to tighten. If the positioning underneath still feels wrong, move to **The New Chapter Workbook** before rewriting again.

---

## Website Project Path

Use this order for a site launch or relaunch.

1. **Before the Session** — name the goal and the real problem.
2. **The New Chapter Workbook** — decide the position the site must argue for.
3. **The Build Copy Guide** — build the page structure and draft each section.
4. **The Brand/voice work inside The Prompt Vault** — use AI only after the brief is clear.
5. **The Edit: DIY Edition** — run copy audit, voice edit, and line edit passes.

Do not skip step two if your business changed. A better homepage cannot fix an old position.

## If Your Messaging Feels Generic

Start with **The Rewrite Playbook**.

Generic messaging is usually not a writing problem first. It is a truth problem. The old sentence is still protecting an older version of you: the version who wanted to be acceptable, flexible, broad, impressive, or impossible to reject.

The Rewrite Playbook helps you rebuild the professional story from the inside out. It covers story audit, positioning, LinkedIn sections, bio lengths, and the through-line that makes a pivot make sense.

After The Rewrite, use **The Edit** to sharpen the draft and **The Prompt Vault** to generate variants without losing the spine.

---

## Messaging Project Path

Use this order when your public language does not match the current work.

1. **Before the Session** — write down what feels misread.
2. **The Rewrite Playbook** — rebuild the story and positioning.
3. **The Byline Method** — capture the voice patterns that make the story sound like you.
4. **The Edit: DIY Edition** — remove hedges, filler, and category language.
5. **The Prompt Vault** — create alternate bio, LinkedIn, and intro versions once the truth is set.

The mistake is asking AI for better language before you have better thinking. The method fixes the thinking first.

## If You Need A Content System

Start with **The Social Strategy Playbook**.

You do not need more content ideas if every week starts from a blank page. You need pillars, a voice snapshot, a platform map, and a rhythm you can keep on your worst week.

The Social Strategy Playbook helps you decide what you are known for, what you repeat on purpose, how the same idea changes by platform, and how to batch a month without making the whole thing feel manufactured.

Pair it with **The Byline Method** when the content needs to sound more like a principal. Pair it with **The Prompt Vault** when you want AI to help draft without turning the voice into oatmeal.

---

## Content Project Path

Use this order for a 30-day content sprint.

1. **The Social Strategy Playbook** — define audience, pillars, and cadence.
2. **The Byline Method** — capture voice from transcripts, calls, or raw notes.
3. **The Prompt Vault** — generate hooks, outlines, and draft options.
4. **The Edit: DIY Edition** — tighten before publishing.
5. **The Rewrite Playbook** — revisit if the content keeps pointing to a bigger positioning problem.

Content strategy is not a mood. It is a repeatable relationship between what you believe, what the audience needs, and what you can keep saying without resenting it.

## If Someone Else Writes For You

Start with **The Byline Method**.

Ghostwriting fails when the writer only captures topics. Voice is not topic. Voice is rhythm, emphasis, worldview, sentence shape, what someone refuses to say, and where the argument naturally lands.

The Byline Method gives you the interview structure, transcript-mining process, tone calibration, and editorial system for writing in a voice without flattening it.

Use it before hiring a writer, briefing a contractor, onboarding a content lead, or training AI. A good voice profile saves more time than another revision round.

---

## Voice Capture Project Path

Use this order when voice is the main asset.

1. Record or gather raw voice: call transcript, interview, email, notes, posts.
2. Use **The Byline Method** to identify cadence, vocabulary, worldview, and hard no's.
3. Use **The Prompt Vault** to turn the voice profile into reusable AI instructions.
4. Use **The Social Strategy Playbook** to decide where that voice shows up.
5. Use **The Edit** to protect the final draft from polish that erases the person.

The goal is not "professional." The goal is recognizable.

## If You Have Drafts But They Feel Soft

Start with **The Edit: DIY Edition**.

Most copy is not bad. It is unedited at the level that matters. The facts are there, but the verbs are weak. The voice is hiding. The sentence says something technically accurate and emotionally useless.

The Edit gives you three passes: copy audit, voice edit, line edit. You check whether the piece is doing the right job, whether it sounds like the right person, and whether every sentence has earned its place.

Use it on bios, pages, emails, posts, proposals, launch copy, and anything else that leaves your hands representing you.

---

## Editing Project Path

Use this order for any drafted piece.

1. **Copy audit:** What is this piece supposed to do?
2. **Reader audit:** Who is it for, and what do they need to believe next?
3. **Voice edit:** Which sentences could belong to anyone?
4. **Line edit:** Which words are hedging, decorating, or delaying?
5. **Final read:** Would I say this out loud without apologizing for it?

If the answer to the final read is no, the draft is not done. It may be polished. It is not ready.

## If The Whole Brand Changed

Start with **The New Chapter Workbook**.

A new chapter is not a new font. It is a new decision about what the brand is here to do, who it is for, what it refuses, and what the site must make unmistakable.

The workbook helps you audit the old brand, map the new position, decide what each page has to say, and rebuild the voice around the business you actually have now.

Use this before a rebrand, relaunch, pivot, price increase, service change, or return after a long quiet season.

## Repositioning Project Path

Use this order when the old story is still on the walls.

1. **Before the Session** — name the decision you are avoiding.
2. **The New Chapter Workbook** — audit old/new fit and write the new position.
3. **The Build Copy Guide** — turn the position into pages.
4. **The Rewrite Playbook** — update the personal story behind the brand.
5. **The Edit** — cut the orphaned old language.

A repositioning is successful when the new promise appears everywhere: homepage, bio, offers, emails, calls, and the way you explain what you are no longer doing.

## If You Need To Think Before A Call

Start with **Before the Session**.

The best strategy sessions begin before the hour starts. If you arrive with a vague problem, the first half of the call becomes excavation. If you arrive with your thinking already on the page, the call can move into decision.

Before the Session helps you name the context, the stakes, the resistance, and the question under the question. It works before a client call, consultant session, brand audit, hiring conversation, or solo planning block.

Use it whenever you are tempted to say, "I do not know where to start."

---

## Strategy Prep Project Path

Use this order when clarity is the deliverable.

1. Fill out **Before the Session** without performing competence.
2. Circle the question that scares you most.
3. Bring that question into the meeting or solo work block first.
4. Use the relevant guide after the session turns the problem into a job.
5. End with one decision, one owner, and one next action.

The question you are embarrassed to ask is often the one that saves the project.

## If You Want AI To Help

Start with **The Prompt Vault**.

AI is not the strategy. The prompt is. A vague prompt produces average language because average is what the machine knows how to do fastest. Your job is to give it context, constraints, voice, judgment, and a job.

The Prompt Vault gives you copy-paste prompts for positioning, bios, website pages, content, emails, and editing. It is built to pair with every other guide in this bundle.

Use AI after you know what you are trying to say. Use it to draft, vary, pressure-test, and compress. Do not use it to decide who you are.

---

## AI Project Path

Use this order when AI is part of the workflow.

1. Use **Before the Session** or **The Rewrite** to clarify the job.
2. Use **The Byline Method** to create a voice profile.
3. Use **The Prompt Vault** to draft in passes.
4. Use **The Edit** to remove generic phrasing and false confidence.
5. Save the final prompt and output as a reusable asset.

If the output sounds like everyone else, the prompt did not contain enough of you.

## The One-Guide Rule

The fastest way to waste The Services Vault is to open eight files and finish none.

Pick one job. Pick one guide. Run the first framework all the way through. Then decide whether the next right guide is upstream or downstream.

Upstream means the issue is more foundational: positioning, clarity, voice, decision.

Downstream means the issue is execution: page structure, captions, edits, prompts.

Most projects fail because people work downstream before the upstream decision is made.
